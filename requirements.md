# 要件定義書（Requirements）

> 固定順デッキ型TCG ブラウザプロトタイプ
> 世界観：星典×星霊（暫定）

---

## 1. プロダクト概要

### 1.1 プロダクト名
固定順デッキ型TCG（仮称）

### 1.2 コンセプト
運要素を極力排除し、**「構築」と「読み合い」に特化した**カードゲーム。
プレイヤーは**星詠み師**として、星の力を記した**「星典（せいてん）」**を持ち、
星を詠むほど力を引き出せるが、星典の星が消えれば術者の運命も尽きる。

**核心的なジレンマ：**
> 星典を詠めば星力（リソース）を得られるが、星典が尽きれば敗北する。
> 「今詠むか、温存するか」の判断が常に勝敗を左右する。

### 1.3 コア体験
- 引き運が存在しない（固定順デッキ）
- 全ては構築段階で決まる
- 「星典をどこまで詠むか」のリスク管理
- 攻撃と防御の読み合いが毎バトルで発生する
- 星霊×星術の**共鳴（レゾナンス）**コンボの発見

### 1.4 ターゲットユーザー
- 思考戦・戦略ゲームが好きなユーザー
- 将棋・チェス・Slay the Spire系が好きな層

## 2. 機能要件

### 2.1 ゲーム形式
| 項目 | 仕様 |
|------|------|
| 対戦形式 | 1対1（プレイヤー vs CPU） |
| ゲーム方式 | ターン制 |
| 実行環境 | ブラウザ上（ローカル完結） |
| サーバー要件 | なし（フロントエンドのみ） |

### 2.2 勝利条件（2つ — いずれかで勝利）
1. **星典消滅**：相手の星典ページが0になる（詠み尽くされる）
2. **星霊全滅**：相手の場に星霊がいなくなり、再召喚もできない

### 2.3 星典（デッキ）仕様
| 項目 | 仕様 |
|------|------|
| 枚数 | 20ページ（MVP） |
| 順番 | 固定（シャッフルなし） |
| 確定タイミング | ゲーム開始時 |
| 可視性 | プレイヤーは自分の星典の全ページ順序を常に確認可能 |
| 1ページ目 | 必ず星霊カード |

**星典の三位一体設計：**
- **デッキ**（カード供給源）
- **リソース生産源**（詠む → 星力獲得）
- **ライフポイント**（尽きたら敗北）

### 2.4 カード種類

#### 2.4.1 星霊カード
- フィールドに配置する戦闘の本体
- **星力値（power）** を持つ（星力衝突で使用）
- **属性** を持つ（共鳴システム用：火・水・風・地）
- 状態管理：**輝態（通常）** → **蝕態（負傷）** → **消星（除去）**
- ダメージを「星護（かばう）」ことができる

#### 2.4.2 星術カード
- 星力を消費して使用する
- **攻星術（攻）**：自分のバトルフェイズで使用、攻撃を仕掛ける
- **守星術（防）**：相手のバトルフェイズで使用、攻撃を防ぐ
- **ダメージ値**：攻撃成功時に相手の星典をめくらせるページ数
- **星力上昇値**：星力衝突時に星霊の星力値に加算される値
- **属性** を持つ（共鳴システム用：火・水・風・地）

#### 2.4.3 星命カード（サポート）
- 即時効果を発動する
- 1ターン1枚まで使用可能
- 効果例：星典回復・バフ・星力チャージ等

### 2.5 リソースシステム（星力：SP）
| 項目 | 仕様 |
|------|------|
| リソース名 | 星力（Star Power: SP） |
| 獲得方法 | 星典を1ページ詠むごとに **SP+2** |
| スタートフェイズ | **0〜3ページ**を任意に詠める |
| エンドフェイズ | **1ページ**自動で詠む |
| 消費 | 星術・星命カード使用時にコスト分消費 |
| 蓄積 | 未使用SPは次ターンに持ち越し可能 |
| 初期値 | 2（ゲーム開始時に星典1ページ詠みで獲得） |

### 2.6 基本ステータス
- **星典残りページ数**：実質的なライフポイント
- **SP（星力）**：星術の使用コスト
- **星典インデックス**：現在の詠み位置
- **場の星霊**：輝態/蝕態の状態

## 3. ゲームフロー要件

### 3.1 ゲーム開始処理
1. 星典（20ページ）をセット
2. 1ページ目の星霊を場に召喚
3. 星典を1ページ詠む → SP+2
4. 先攻/後攻決定（MVPではプレイヤー先攻固定）

### 3.2 ターン処理フロー
```
スタートフェイズ（詠みフェイズ）
  └─ 星典を0〜3ページ任意に詠む（1ページにつきSP+2）
     ※ジレンマ：SPが欲しい vs 星典を温存したい

メインフェイズ
  ├─ 現在の天窓（見開き）のカードを使用可能
  ├─ 星霊カード → 場に召喚
  └─ 星命カード → 即時効果発動（1ターン1枚）

バトルフェイズ
  ├─ 1. 星撃宣言（攻撃側）
  ├─ 2. 攻撃側：攻星術を使用（SP消費）
  ├─ 3. 防御側：守星術で対応 or パス
  ├─ 4. 星力衝突
  │     攻撃側合計 = 星霊の星力値 + 星術の星力上昇値 [+ 共鳴ボーナス]
  │     防御側合計 = 星霊の星力値 + 星術の星力上昇値 [+ 共鳴ボーナス]（未使用なら0）
  ├─ 5. 判定
  │     攻撃側 > 防御側 → 攻撃成功 → ダメージ処理
  │     攻撃側 ≤ 防御側 → 攻撃失敗（ノーダメージ）
  └─ 6. ダメージ解決
        └─ 星術のダメージ値分、相手の星典をめくらせる
           or 相手の星霊が「星護」（星霊が蝕態に）

エンドフェイズ
  ├─ 場の整理（消星した星霊の除去）
  ├─ 星典1ページ自動詠み → SP+2
  └─ 勝敗判定
```

### 3.3 星護（かばう）システム
- 星典へのダメージを、場の星霊が肩代わりできる
- **輝態の星霊** → 星護 → **蝕態に変化**
- **蝕態の星霊** → 星護 → **消星（除去）**
- 星護するかどうかはプレイヤーの選択

### 3.4 星力衝突の詳細
- 攻撃側が星術を使用 → 防御側が対応する機会を得る
- 防御側が星術を使わない場合、防御側星力 = 0
- 攻撃側合計 > 防御側合計 で攻撃成功
- 同値は防御側有利（攻撃失敗）

## 4. 独自システム要件

### 4.1 共鳴（レゾナンス）システム
- 星霊と星術が**同じ属性**の場合、星力衝突で **+1 ボーナス**
- 例：火属性の星霊 × 火属性の攻星術 → 合計星力 +1
- 防御でも同様：水属性の星霊 × 水属性の守星術 → 合計星力 +1
- デッキ構築時に「属性の組み合わせ」を考える戦略レイヤーを追加

### 4.2 過詠（オーバーチャージ）システム
- 星術使用時、通常コスト + **追加SP** を支払うことで効果を強化できる
- **追加SP 2ごとにダメージ +1**（攻星術の場合）
- **追加SP 2ごとに星力上昇 +1**（守星術の場合）
- 上限：通常コストの2倍まで（例：コスト2の術→最大追加SP2→合計4SP）
- リスク：SPを大量消費 = 星典を多く詠む必要 = ライフが削れる

## 5. CPU（AI）要件

### 5.1 基本方針
- シンプルなルールベースAI
- 「それっぽく動く」ことを優先
- 完全最適化は不要

### 5.2 行動アルゴリズム
1. **スタートフェイズ**：状況に応じて詠み枚数を決定（SP残量・残りページ考慮）
2. **メインフェイズ**：使用可能な星霊・星命を優先的にプレイ
3. **バトルフェイズ**：
   - 攻撃：共鳴する攻星術を優先選択
   - 防御：守星術で防げそうなら使用
   - 過詠：SPに余裕があれば使用
4. **星護判断**：星典残りページが少ない場合は星霊で星護

### 5.3 CPUデッキ
- 固定の事前定義デッキを使用
- プレイヤーとは異なる構成

## 6. UI要件

### 6.1 必須画面要素
| 要素 | 説明 |
|------|------|
| 星典エリア | 残りページ数の表示、詠みボタン |
| 天窓エリア（見開き） | 現在の見開きページのカードを表示 |
| フィールドエリア | プレイヤー/CPUの星霊を表示（輝態/蝕態の視覚表現） |
| SP表示 | 現在の星力 |
| 残りページ表示 | 両者の星典残りページ数（実質ライフ） |
| コントロールボタン | 詠みボタン・星撃宣言・ターンエンド |
| ログエリア | 行動の履歴を表示 |
| 星典プレビュー | 自分の星典の全ページ順序を確認 |

### 6.2 インタラクション
- **詠みボタン** → 星典を1ページ詠む（スタートフェイズ中、最大3回）
- **カードクリック** → カード使用（星霊召喚 or 星術使用 or 星命使用）
- **星霊クリック** → 攻撃 or 星護選択
- **星撃宣言ボタン** → 戦闘開始
- **ターンエンドボタン** → ターン終了
- **過詠スライダー** → 追加SP量の選択
